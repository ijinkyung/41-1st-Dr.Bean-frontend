## 🥹 회고
### 잘한 점
우리 팀이 잘한 점은 처음부터 무리하지 않았다는 점이다. 구현하고 싶은 기능은 많았지만 처음부터 욕심부리다가 프로젝트를 완성하지 못하는 대참사가 벌어질까봐 agile 방식으로 할 수 있는 기능부터 차근차근 기획해나갔다.

첫 Planning Meeting 때 커피로 주제를 변경하고 구현하고 싶은 기능을 추가했다.
닥터마틴의 **지속가능성**이라는 브랜드 가치와 커피의 지속가능성을 결합해 브랜딩을 진행하였다.

### 아쉬운 점
첫 미팅때 **공통적**으로 사용할 component나 css 요소들을 자세히 나눠서 코드 컨벤션을  기획했어야했다!
겹치는 부분이 생길때마다 pr을 올려 공통요소를 추가하다보니 시간이 리팩토링하는 시간이 길어진 부분에서 아쉬다고 느껴졌다.

다른 **팀원들의 코드**를 자세히 보지 못했다는 점도 아쉬웠다. 
첫주에는 내가 맡은 기능을 구현하는 것에 몰두하다 보니까 다른 팀원분들이 어떤 식으로 코드를 짜는지 볼 정신이 하나도 없었다. 
다음부터는 다른 팀원들의 코드도 보면서 나와 다르게 구현한 기능도 참고하고 적용시켜봐야겠다.

### 마지막 소감
1. useParams, seachParams에 대해 배울 수 있어서 좋았다.
2. 리팩토링은 필수이다. 코드를 다시보니 주석 안지운것도 많고, 이제 와서 보니 더 간결하고 가독성있게 작성할 수 있는 코드들이 많다. 설날에 리팩토링이랑 추가구현을 완료해야지!
